# Help and Frequently Asked Questions

This section collects answers to common questions about using Playlite, along with important guidance on data security.

If you do not find your answer here, check the getting started guide or the project's additional documentation.

## Steam Library Import

### Steam import does not work. What should I check?

Before anything else, verify the points below:

- Your **SteamID** is in **64-bit** format
- Your **Steam Web API Key** is valid
- Your profile and library are visible (depending on Steam privacy settings)
- You are connected to the internet during the import

For a complete step-by-step guide, see the Steam import guide:

- [Importing your Steam library](/en/guide/getting-started#2-importing-your-library-steam)

## Game Metadata and Information

### Why is some data incomplete (for example, genre)?

Not all information is available directly during the initial import.

In those cases, Playlite imports the essential data and lets you complete or correct the information later.

Some data, such as genre and more complete descriptions, can be obtained later:

- by using the **Fetch Metadata** option in Settings
- by editing the game manually, especially if it is not from your Steam library

## Backup and Data Restore

Playlite offers a simple **backup and restore** system to keep your data safe.

All backups are generated in **JSON** format, making storage and recovery easy.

![Backup](/images/app-settings-backup.png)

### How to back up (export data)

1. Open **Settings** in the app.
2. Find the **Backup / Export Data** option.
3. Choose where to save the file.
4. Confirm the operation.

### How to restore a backup (import data)

1. Open **Settings**.
2. Find the **Restore / Import Data** option.
3. Select the backup file (JSON).
4. Confirm and wait for the process.
5. Check **Libraries** to make sure the data was restored correctly.

### Security best practices

- Back up before major changes (imports, reorganizations, or tests).
- Keep your backup files in a safe place (for example, a dedicated folder, the cloud, or external media).
- Avoid overwriting old backups without keeping an alternative copy.

### If something goes wrong

- Check whether the selected file is a JSON file generated by Playlite.
- If the restore fails, consider:
  - backing up the current state before trying again
  - restoring on a clean installation of the app

## Known limitations

Playlite uses the RAWG API to enrich games with covers and descriptions automatically.
Indie games, especially those distributed outside official stores like Steam,
may not exist in the database or may have generic names.

In those cases, the app may:

- fail to find information
- or associate incorrect data with games that have similar names

When that happens, we recommend editing the game manually to correct the information
or add the missing data.
